Mini-Glossar

Datenbank: Eine strukturierte Sammlung von Daten – zum Beispiel eine Tabelle mit Schülernamen, Klassen und Noten.

Datenbankmanagementsystem (DBMS): Das Programm, das die Datenbank verwaltet. Es speichert, sucht und schützt die Daten. Beispiele: MySQL, PostgreSQL, SQLite.

SQL (Structured Query Language): Die Sprache, mit der man dem DBMS Anweisungen gibt – also sagt, welche Daten man sehen, einfügen oder löschen möchte.

Tabelle: Eine geordnete Form, um Daten darzustellen. Zeilen heißen Datensätze, Spalten heißen Felder oder Attribute.

WHERE: Ein Teil eines SQL-Befehls, der eine Bedingung festlegt. Beispiel: „Zeig mir nur die Zeilen, wo der Name gleich alice ist.“

Phase 1: Normale Abfrage

1a) Tabelle lesen

Suche dir einen Namen aus der Tabelle aus. Beispiel: alice.

1b) Abfrage in Worten

„Hol alle Spalten * aus der Tabelle students, aber nur die Zeilen, wo username gleich alice ist.“

SELECT * FROM students WHERE username = 'alice';

1c) Abfrage ausführen

Gut. Du hast eine normale Abfrage erfolgreich ausgeführt.

Phase 2: Manipulation ausprobieren

Jetzt geben wir etwas ein, das keine Person ist. Teste ' OR '1'='1' --. Das soll alle Zeilen liefern.

Generierte Query (unsicher, String-Konkatenation) 
SELECT * FROM students WHERE username = '…';

Die Eingabe wird Teil der SQL-Zeichenkette. Dadurch kann sie die Bedingung verändern.

Richtig. Alle Zeilen wurden angezeigt. Das ist die Wirkung einer SQL-Injection.

Phase 3: Warum funktioniert das

  1. Die Anwendung setzt deine Eingabe ungeschützt in die SQL-Zeichenkette.
  2. In der Eingabe steckt eine OR-Bedingung, die immer wahr ist.
  3. Damit ist die WHERE-Klausel immer wahr → die Datenbank gibt alle Zeilen zurück.
So sieht die manipulierte Abfrage aus 
SELECT * FROM students WHERE username = '' OR '1'='1';

Der grün markierte Teil kommt aus der Eingabe und ändert die Logik.

Phase 4: Sichere Variante (Parameter)

Jetzt dieselbe Eingabe, aber die Anwendung nutzt Parameterbindung. Die Struktur bleibt gleich.

Generierte Query (sicher, Parameter) 
SELECT * FROM students WHERE username = ?  -- bind: ['alice']

Die Eingabe ist ein Wert, kein Code. Eine OR-Bedingung landet nicht in der Struktur.

Gut. Im sicheren Modus erscheinen nicht alle Zeilen. So muss es sein.

Aufgaben (kurz)

  1. Phase 2: Notiere die Eingabe, die alle Zeilen zeigt.
  2. Erkläre in einem Satz, warum diese Eingabe wirkt.
  3. Phase 4: Was ändert sich und warum?
  4. Nenne zwei Schutzmaßnahmen.

Lehrkraft: Lösungen und Hinweise

  • Muster-Input: ' OR '1'='1' --
  • Kernaussage: OR-Bedingung macht WHERE immer wahr.
  • Sicher: Prepared Statements, Input-Validation, least privilege.
